Introduction

Adminca is an perfect, powerful, fully customizable and very easy to use universal admin template that is suitable for any type of web application. Based on the popular Bootstrap 4framework, it provides a completely adaptive and perfect interface. A comfortable structure and high-quality code will make it very easy to use a template that can easily be customized even a novice developer. Adminca comes with several versions of templates: HTML and Angular 5+.
Adminca: Save your time, choose the best, we work a lot to make you comfortable.

Quick Start

If you want to change something in the template using compilation, run the following 6 commands.

  1. Download and install the latest Node.js.
  2. Install bower globally: 
    npm install bower -g
  3. Install gulp globally: 
    npm install gulp-cli -g
  4. Navigate to [adminca_folder]/tools/gulp folder: 
    cd tools/gulp
  5. Install necessary tools to build your project. Must execute in [adminca_folder]/tools/gulp folder: 
    npm install
  6. Compile all the themes. At the output we get our compiled themes in the [adminca_folder]/dist folder. 
    gulp build

    If you want to compile one theme, run the command gulp build:[theme]. [theme] == one of the themes:

    • admin_1 // gulp build:admin_1
    • admin_2
    • admin_3
    • ... // other templates
    Read more about the compilation in section Gulp build.

Gulp build

Installing

Gulp is a toolkit for automating painful or time-consuming tasks in your development workflow. gulpjs.com
To install and use Gulp, you will need Node.js installed to your computer.

1) To install Gulp globally, run the following command on the command line:

npm install gulp-cli -g

2) And install Bower globally:

npm install bower -g

3) Then navigate to [root]tools/gulp directory and run

npm install

This command will create a node_modules directory and install gulp and all the necessary tools to build your project.

Tasks

The entire task gulp is in [root]/tools/gulp/gulp-tasks folder.

We have built for you a very convenient and flexible build system that even a beginner can easily understand.
You can easily create as one theme, and all together in one command.

Let's start. Since we have several themes: admin_1, admin_3, Admin_3 ..., we have accordingly created a separate task for each themes

gulp build:admin_1  //== BUILD FILES FOR ADMIN_1 THEME
    gulp.start('admin_1:pug');     // BUILD HTML FILES FOR ADMIN_1
    gulp.start('admin_1:sass');    // BUILD CSS FILES FOR ADMIN_1
    gulp.start('admin_1:js');      // BUILD JS FILES FOR ADMIN_1

gulp build:admin_2  //== BUILD ALL FILES FOR ADMIN_2 THEME
    gulp.start('admin_2:pug');     // BUILD HTML FILES FOR ADMIN_2
    gulp.start('admin_2:sass');    // BUILD CSS FILES FOR ADMIN_2
    gulp.start('admin_2:js');      // BUILD JS FILES FOR ADMIN_2

gulp build:admin_3  //== BUILD ALL FILES FOR ADMIN_3 THEME
    ...

so we see that for the assembly of Admin_1 theme we need one command: gulp build:admin_1, which creates all files for this theme.

What it does in detail:

  • run gulp.start('admin_1:pug'); command which will take all the pug files from the [root]/src/pug/admin_1/pages/*.pug folder specified in the build.admin_1.pug.src config parameter.
    • Compiles them into html files.
    • And will save them in the [root]/dist/admin_1/html folder specified in the build.admin_1.pug.output config parameter.
  • run gulp.start('admin_1:js'); command which will take
    • [root]/src/js/app/app.js
    • [root]/src/js/theme/admin_1/layout.js
    files specified in the build.admin_1.pug.src config parameter.
    • Minify.
    • Then concat them in a app.js file.
    • And will save it in the [root]/dist/admin_1/html/assets/js folder specified in the build.admin_1.js.output config parameter.
  • run gulp.start('admin_1:sass'); command which will take
    • [root]/src/sass/components/components.scss -- ui components
    • [root]/src/sass/layouts/admin_1/layout.scss -- admin_1 theme layout
    • [root]/src/sass/vendors/vendors.scss -- reset vendor plugins styles
    files specified in the build.admin_1.sass.src config parameter.
    • Concat them in a main.css file.
    • Minify.
    • And will save it in the [root]/dist/admin_1/html/assets/css folder specified in the build.admin_1.sass.output config parameter.

    And compiles the rest of the sass files.

  • Creates the rest of the files: pictures, vendor plugins...

So at the output we get such a folder in the output folder [root]/dist/admin_1/html

  • assets
    • css
      • main.css
      • themes
      • pages
    • js
      • app.js
      • scripts
    • img
    • vendors
    • demo
  • index.html
  • dashboard_2.html
  • dashboard_3.html
  • ...

Stylesheets

Overview

Each template in Adminca includes one main style: main.min.css file, consisting of:

  • styles for ui components
  • a layout styles
  • styles for vendor plugins

The styles for ui components are compiled from [root]/src/sass/components/components.scss file.

Layout styles are compiled from the [root]/src/sass/layouts/[theme]/layout.scss file.

Styles for vendor plugins are compiled from [root]/src/sass/vendors/vendors.scss file.

They are combined in one main.min.css file for each version of the template.

So we get

[Admin_1] theme[Admin_2] theme[Admin_3] theme
sass/components/components.scss sass/components/components.scss sass/components/components.scss
sass/layouts/admin_1/layout.scss sass/layouts/admin_2/layout.scss sass/layouts/admin_3/layout.scss
sass/vendors/vendors.scss sass/vendors/vendors.scss sass/vendors/vendors.scss
so we get
main.cssmain.cssmain.css

Scotch colors

Pay attention to the $scotch-colors array. For each color in this array will be created a palette of background from 50-800 if you specify the parameter palette: true. for example:


// COLORS PALLETES
//
// The colors for which the palette will be created

$scotch-colors: (
    'blue': (
        color: $blue,
        palette: true,
    ),
);

Installation

The Angular CLI is a command line interface tool that can create a project, add files, and perform a variety of ongoing development tasks such as testing, bundling, and deployment.

  1. Download and install the latest Node.js, if you have not already installed it.

  2. Install the Angular CLI globally: Open the console and run the following command:

    npm install -g @angular/cli

  3. Setup Angular app: navigate into angular app folder: [root]/src/angular/[theme] and run the following command:

    npm install

This command will install all the required Node.js modules into the node_modules directory inside your angular app folder.

Server

Launch the server. The ng serve command launches the server, watches your files, and rebuilds the app as you make changes to those files. Using the --open (or just -o) option will automatically open your browser on http://localhost:4200.

ng serve --open
Build

The ng build command compiles the application into an output directory dist.

ng build --prod

Detailed information about the build can be found in the official documentation.
