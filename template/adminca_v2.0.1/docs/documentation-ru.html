Введение

Adminca - это идеальный, мощный, полностью настраиваемый и очень простой к использованию универсальный шаблон администратора который подойдет для любых типов веб-приложений. Основывая на популярном фреймворке Bootstrap 4 он обеспечивает полностью адаптивный и совершенный интерфейс. Удобная структура и высококачественный код сделают его очень простой к использованию шаблоном который может легко настроиват даже начинающий разработчик. Adminca поставляется с несколькими версиями шаблонов: HTML и Angular 5+.
Adminca: Сэкономьте свое время, выберите лучшее, ведь мы много работаем чтобы вам было удобно.

Быстрый старт

Если вы хотите что то изменить в шаблоне с помощью компиляции, выполните следуюшие 6 команды.

  1. Загрузите и установите последнюю версию Node.js.
  2. Установите bower глобально: 
    npm install bower -g
  3. Установите gulp глобально: 
    npm install gulp-cli -g
  4. Перейдите в [adminca_folder]/tools/gulp папку: 
    cd tools/gulp
  5. Установите инструменты для сборки вашего проекта. Необходимо выполнить в папке [adminca_folder]/tools/gulp: 
    npm install
  6. Скомпилируйте все темы. На выходе мы получаем наши скомпилированные темы в папке [adminca_folder]/dist. 
    gulp build

    Если вы хотите скомпилировать одну тему выполните команду gulp build:[theme]. [theme] == один из тем:

    • admin_1 // gulp build:admin_1
    • admin_2
    • admin_3
    • ... // остальные шаблоны
    Подробно про компиляцию читайте в секцию Gulp сборка.

Gulp сборка

Установка

Gulp - это инструментарий для автоматизации трудоемких задач в процессе разработки. gulpjs.com
Чтобы установить и использовать Gulp, вам понадобится Node.js установленный на вашем компьютере.

1) Чтобы установить Gulp глобально выполните следующую команду в командной строке:

npm install gulp-cli -g

2) И установим Bower глобально:

npm install bower -g

3) Потом перейдите в каталог [root]tools/gulp и выполните

npm install

Эта команда создаст каталог node_modules и установит в нем gulp и все необходимые инструменты для сборки вашего проекта.

Таски

Все gulp таски находится в [root]/tools/gulp/gulp-tasks папке.

Мы создали для вас очень удобную и гибкую систему сборки, который легко может понять даже начинающий.
Вы можете легко создать как одну тему, так и все вместе в одной команде.

Давайте начнем. Поскольку у нас есть несколько тем: admin_1, admin_2, Admin_3 ..., поэтому мы создали отдельные задачи для каждой темы.

gulp build:admin_1  //== СОБРАТЬ ВСЕ ФАЙЛЫ ДЛЯ ADMIN_1 ТЕМ
    gulp.start('admin_1:pug');     // СОБРАТЬ HTML ФАЙЛЫ ДЛЯ ADMIN_1 ТЕМ
    gulp.start('admin_1:sass');    // СОБРАТЬ CSS ФАЙЛЫ ДЛЯ ADMIN_1 ТЕМ
    gulp.start('admin_1:js');      // СОБРАТЬ JS ФАЙЛЫ ДЛЯ ADMIN_1 ТЕМ

gulp build:admin_2  //== СОБРАТЬ ВСЕ ФАЙЛЫ ДЛЯ ADMIN_2 ТЕМ
    gulp.start('admin_2:pug');     // СОБРАТЬ HTML ФАЙЛЫ ДЛЯ ADMIN_2 ТЕМ
    gulp.start('admin_2:sass');    // СОБРАТЬ CSS ФАЙЛЫ ДЛЯ ADMIN_2 ТЕМ
    gulp.start('admin_2:js');      // СОБРАТЬ JS ФАЙЛЫ ДЛЯ ADMIN_2 ТЕМ

gulp build:admin_3  //== СОБРАТЬ ВСЕ ФАЙЛЫ ДЛЯ ADMIN_3 ТЕМ
    ...

Так мы видим, что для сборки Admin_1 тем нам нужна одна команда: gulp build:admin_1, который создает все файлы для этой темы.

Подробно что он делает:

  • запускает gulp.start('admin_1:pug'); команду который возьмет все pug файлы из [root]/src/pug/admin_1/pages/*.pug папки указанный вbuild.admin_1.pug.src параметре конигурации.
    • Компилирует их в html файлы.
    • И сохранит их в [root]/dist/admin_1/html папке указанный в build.admin_1.pug.output параметре конигурации.
  • запускает gulp.start('admin_1:js'); команду который возьмет
    • [root]/src/js/app/app.js
    • [root]/src/js/theme/admin_1/layout.js
    файлы указанный в build.admin_1.pug.src параметре конигурации.
    • Минифицировать.
    • Затем соединит их в app.js файле.
    • И сохранит его в [root]/dist/admin_1/html/assets/js папке указанный в build.admin_1.js.output параметре конигурации.
  • запускает gulp.start('admin_1:sass'); команду который возьмет
    • [root]/src/sass/components/components.scss -- ui компоненты
    • [root]/src/sass/layouts/admin_1/layout.scss -- стили для макет темы admin_1
    • [root]/src/sass/vendors/vendors.scss -- стили для вендорных плагинов
    файлы указанный в build.admin_1.sass.src параметре конигурации.
    • соединит их в main.css файле.
    • Минифицировать.
    • И сохранит его в [root]/dist/admin_1/html/assets/css папке указанный в build.admin_1.sass.output параметре конигурации.

    И компилирует остальные sass файлы.

  • Создает остальные файлы: картинки, вендорные плагины...

На выходе мы получаем такую папку в выходном папке [root]/dist/admin_1/html

  • assets
    • css
      • main.css
      • themes
      • pages
    • js
      • app.js
      • scripts
    • img
    • vendors
    • demo
  • index.html
  • dashboard_2.html
  • dashboard_3.html
  • ...

Изменения файлов

Gulp может запускаться автоматически при изменении файла.

gulp serve

Это команда запускает сервер с помощью browserSync и отслеживает в реальном времени все изменения с исходными файлами в папке src и в зависимости от изменений выполнять конкретный таск.

Стили

Обзор

Каждый шаблон в Adminca включает один главный стиль: main.min.css файл, состоящий из:

  • стили для ui компонентов
  • стили макета
  • стили для вендорных плагинов

Стили для ui компонентов компилируются из [root]/src/sass/components/components.scss файла.

Стили макета компилируются из [root]/src/sass/layouts/[theme]/layout.scss файла.

Стили для вендорных плагинов компилируются из [root]/src/sass/vendors/vendors.scss файла.

Они соединяются в одном файле main.min.css для каждого версии шаблона.

Таким образом мы получаем

[Admin_1] theme[Admin_2] theme[Admin_2] theme
sass/components/components.scss sass/components/components.scss sass/components/components.scss
sass/layouts/admin_1/layout.scss sass/layouts/admin_2/layout.scss sass/layouts/admin_3/layout.scss
sass/vendors/vendors.scss sass/vendors/vendors.scss sass/vendors/vendors.scss
в итоге получаем
main.cssmain.cssmain.css

Scotch colors

Обращайте внимание на $scotch-colors массив. Для каждого цвета в этом массиве будет создана палитра фона от 50-800 если задать параметр palette: true. Например:


// COLORS PALLETES
//
// The colors for which the palette will be created

$scotch-colors: (
    'blue': (
        color: $blue,
        palette: true,
    ),
);

Установка

Angular CLI - это инструмент интерфейса командной строки, который может создавать проект, добавлять файлы и выполнять множество текущих задач разработки, таких как тестирование, комплектация и развертывание.

  1. Загрузите и установите последнюю версию Node.js, если он у вас еще не установлен.

  2. Установите Angular CLI глобално: Откройте консоль и запустите следующую команду:

    npm install -g @angular/cli

  3. Установите приложение Angular: перейдите в папку приложении Angular: [root]/src/angular/[theme] и запустите следующую команду:

    npm install

Эта команда установит все необходимые модули Node.js в каталог node_modules внутри папки вашей Angular приложении.

Server

Запустите сервер. Команда ng serve запускает сервер, наблюдает за вашими файлами и перестраивает приложение при внесении изменений в эти файлы. Использование опции --open (или просто -o) автоматически откроет ваш браузер на http://localhost:4200.

ng serve --open
Сборка

Команда ng build компилирует приложение в выходной каталог dist.

ng build --prod

Подробную информацию о сборке можно найти в официальном документацию.
